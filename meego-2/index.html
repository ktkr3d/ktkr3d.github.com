MeeGo の種類

  • MeeGo for Netbooks

    • meego.com 版

ファイルシステムはext3, btrfs 日本語入力メソッドはSCIM-SKK 32bit Intel Atom or Intel Core 2 CPU (SSSE3) 前提

* openSUSE 版（Smeegol ）

ファイルシステムはext4 日本語入力メソッドはSCIM-Anthy x86(32bit, 64bit) パッケージ多め(openSUSE のリポジトリ利用可)

* Fedora 版

  • MeeGo Tablet

    • meego.com 版

    • Intel 版

MeeGo for Netbooks の設定

MeeGo Tablet の設定

  • マウスポインタを表示 /etc/sysconfig/uxlaunch でxopts=-nocursor をコメントアウト

    xopts=-nocursor

  • Inspiron duo でタッチスクリーンを有効化 GRUB でカーネルパラメータに以下を追加 usbhid.quirks=0x0eef:0x725e:0×40 vt.handoff=7

  • Inspiron duo でイヤフォンを有効化（ビルドに依存?） /etc/modprobe.d/dist.conf に以下を追加 options snd-hda-intel model=ideapad

  • 隠しアクセスポイントへの接続 su iwconfig Wi-Fiの設定画面でパスフレーズ入力

  • Firefox のインストール 公式からダウンロード $HOME/.local/share/applications/firefox.desktop 作成

  • Thunderbird のインストール 公式からダウンロード $HOME/.local/share/applications/thunderbird.desktop 作成

  • Songbird のインストール 公式からダウンロード $HOME/.local/share/applications/songbird.desktop 作成

  • Music ffmpeg でogg に変換して、$HOME/音楽/ に格納

  • Video ffmpeg2theora でogv に変換して、$HOME/ビデオ/ に格納

  • 環境A はopenSUSE 11.3 版(Smeegol Linux 1.0.3)
  • 環境B はopenSUSE 11.4 版(Milestone 3 + MeeGo UI 1.0)

各パネル用キーボードショートカットの作り方

詳しくはhttp://forum.meego.com/showthread.php?t=619 の#1〜#4 1. [設定]–[キーボード]–[キーボード・ショートカット]を起動 2. [追加]で以下を設定

名前: myzone コマンド: dbus-send –dest=org.moblin.UX.Shell.Toolbar /org/moblin/UX/Shell/Toolbar org.moblin.UX.Shell.Toolbar.ShowPanel string:”moblin-panel-myzone”

  1. 追加された行を選択して、マッピングしたいキーを指定

  3. パネル名string は/usr/share/mutter-moblin/panels/ 下の実ファイル名を参考に

ログアウト/シャットダウン/再起動ダイアログ

gnome-session-save 系のインタフェースが使えないので… 1. gnome-session-shutdown.desktop 等をコピー 2. zenity でダイアログ表示とアクション制御の簡単なシェルスクリプトファイルを作成して、.desktop ファイルのExec= 行に指定する

pick=$(zenity –list –title “Session” –text “Please choose an action.” –radiolist –hide-column 2 –column Pick –column dummy –column Action true 0 “Logout” false 1 “Shut down” false 2 “Restart”);

  • デフォルトにしたいAction をtrue に、その他をfalse に
  • $?(0: OK, 1: CANCEL), $pick(0: Logout, 1: Shut down, 2: Restart)で判定
  • 強制ログアウト: /usr/bin/killall uxlaunch
  • 強制シャットダウン: /sbin/shutdown -h now (visudo しておく)
  • 強制リブート: /sbin/shutdown -r now (visudo しておく)
  • キーボードショートカットにも登録しておくと便利
  • Icon= 行にyast-bootloader を指定する
  • ~/.local/share/favourite-apps に.desktop ファイルのパスを追加 session dialog

アプリケーションウィンドウの最大化解除

どうしてもドラッグ&ドロップしたい時など… 1. タイトルバーをドラッグしてウィンドウをずらす smeegol

ブートローダに他のOS を追加

追加したいOS のファイルシステムをマウントして内容(/etc/default/grub とか)を元に転記するのが楽 1-a. /boot/grub/menu.lst にtitle, root, kernel, initrd を追加 1-b. /boot/grub/menu.lst にtitle, rootnoverify, chainloader を追加 1-c. [YaST]–[ブートローダ]で設定

自動起動するアプリの登録

  1. gnome-session-properties
  2. 既定のサービスの有効/無効化の他に独自スタートアップスクリプトを登録

MyZone が表示されなくなったんだけど…

moblin-panel-myzone プロセスの状態がおかしくなっているので… 1. killall moblin-panel-myzone その後、自動で別プロセスとなり、正常にMyZone が表示できるようになる * 自動で再表示したい場合はDBus でメッセージを飛ばす

デスクトップ環境をGNOME に切り替える

gdm が動いているので普通にログインしなおせばGNOME を選択できる ウィンドウマネージャをtwm, IceWM にすることできる KDE デスクトップのメタパッケージ導入でKDE も選択できると思う meego gnome

  • 起動時に選択したい場合は自動ログインを外しておく [YasT]–[ユーザとグループの管理]の[熟練者用オプション]–[ログイン設定]

MP4(H.264/AAC)動画の再生

1-a. smplayer, vlc 等のプレイヤーをインストールする 1-b. [YaST]–[ソフトウェアリポジトリ]で「Packman」を追加して、gstreamer-0_10 系のプラグインをPackman のものにするとBanshee, Totem で再生できる 1-c. Chromium ブラウザでローカルファイルを直接開く

Songbird で再生する楽曲を通知する

  1. ubuntu-notifyOSD アドオンをインストールすると右下に通知ポップアップが表示されるようになる
  2. install.rdf は適宜修正
  3. chrome.manifest でsongbird.xul をlayoutWithBrowserOverlay.xul に置換
  4. Songbird 1.9 で動かない場合はchmod 755 donotify.py する(本来はchrome フォルダ下をjar 化するんだろうけど面倒なので…)
  5. python-imaging を入れてもアルバムアートは表示されなかった(GNOME 上では表示される) smeegol songbird notify

Bansee がDAAP サーバを自動検知しない

  1. 「編集」-「設定」の「機能拡張」タブでDAAP プラグインのチェックボックスを一旦OFF してON すると検知が始まる
  2. 曲リストが二重に取得されたり、再生が中断する場合は一旦切断してから再接続する
