Inspiron Duo + Android-x86 4.0 でヘッドフォン

2012年03月17日

Inspiron duo + Android-x86 4.0 の環境でヘッドフォンジャックにプラグを挿した時に内部スピーカーをミュートするように設定してみた。 普通のLinux カーネルではalsa-base.conf などにmodel=ideapad を指定すれば良いけれど、Android-x86 では./kernel/sound/pci/hda/patch_realtek.c にmodel のプリセットが定義されているみたいだ。

[cpp firstline=“18470” highlight=“18478” title=“./kernel/sound/pci/hda/patch_realtek.c”] / * configuration and preset / static const char * const alc662_models[ALC662_MODEL_LAST] = {

    [ALC662_3ST_2ch_DIG]    = "3stack-dig",
    [ALC662_3ST_6ch_DIG]    = "3stack-6ch-dig",
    [ALC662_3ST_6ch]        = "3stack-6ch",
    [ALC662_5ST_DIG]        = "5stack-dig",
    [ALC662_LENOVO_101E]    = "lenovo-101e",
    [ALC662_ASUS_EEEPC_P701] = "eeepc-p701",
    [ALC662_ASUS_EEEPC_EP20] = "eeepc-ep20",

[/cpp]

ideapad に相当する指定は、lenovo-101e みたいなので、 以下を設定する。

[plain title=“./device/tegatech/tegav2/tegav2_info”] SNDDEV=“snd-hda-intel model=lenovo-101e position_fix=1 enable=yes” [/plain]
