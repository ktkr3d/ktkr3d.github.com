Plasma/Active/Contour を試してみる

2011年04月14日

http://community.kde.org/Plasma/Active/Installation http://community.kde.org/Plasma/Active/Contour/Installation Plasma Active Tablet パッケージのContour がおもしろそうなのでInspiron duo で試してみる。

  1. openSUSE 11.4 をインストール http://software.opensuse.org/114/ja KDE 版のISO の拡張子を.img にしてsudo image-writer でUSBに入れて、Live USB を作った。

  2. Plasma/Active をインストール（不要だったかも）

/etc/zypp/zypp.conf

solver.allowVendorChange = true

[shell gutter=“false”]

zypper ar http://download.opensuse.org/repositories/home:/rwooninck:/UNSTABLE_KDE4/openSUSE_11.4/ kde-unstable

zypper mr -p 100 kde-unstable

zypper ar http://download.opensuse.org/repositories/home:/vizzzion/kde-unstable/ plasma-active

zypper mr -p 101 plasma-active

zypper refresh -f

zypper in plasma-active:kdebase4-runtime plasma-tablet-config plasma-declarative-widgets

[/shell]

  1. Contour パッケージのインストール http://software.opensuse.org/ でcontour を1-クリック インストール

  2. Contour の自動起動設定 http://software.opensuse.org/ でplasma-contour-config を1-クリック インストール (plasma-tablet プロセスが残っていたらkquitapp plasma-tablet で消しておく)

おもしろそう！！使い方がよくわかんない！！

GNOME Shell を拡張してみた

2011年04月09日

https://wiki.archlinux.org/index.php/GNOME_3 https://live.gnome.org/GnomeShell/CheatSheet http://www.webupd8.org/2011/04/gnome-shell-extensions-additional.html http://git.gnome.org/browse/gnome-shell-extensions/

GNOME Shell Extensions やGNOME 3 の便利な機能を使ってみよう。

GNOME Shell Extensions

[shell gutter=“false”] $ git clone git://git.gnome.org/gnome-shell-extensions $ cd gnome-shell-extensions $ ./autogen.sh $ make $ sudo make install [/shell]

  • alternate-tab: Alt+Tab 操作関連

  • alternative-status-menu: 電源オフメニュー常時表示 標準ではステータスメニューを開いて、「Alt」ボタンを押すと「サスペンド」が「電源オフ…」に切り替わる。alternative-status-menu を使うと「サスペンド」と「電源オフ」の両方が常に表示されるようになり、「Alt」ボタンを押すと「サスペンド」が「Hibernate」に切り替わる。必須!!

  • dock: 右端にドックバーを表示（画面が狭くなった…微妙?）

  • windowsNavigator: overview 画面上でALT+数字キーによりウィンドウ選択

GNOME 3

  • テーマのカスタマイズ gnome-tweak-tool またはsetting.ini

  • スクリーンキャストを撮る Ctrl+Alt+Shift+R で開始&終了。WEBM 形式で出力される。

  • キー操作でoverview へ切り替える システム(Windows)キーまたはAlt+F1

Ubuntu 11.04 でGNOME Shell

2011年04月08日

http://www.ubuntuupdates.org/ppa/gnome_3?dist=natty

Ubuntu 11.04 用のGNOME Shell を含むppa を見つけたので、早速インストールしてみた。ちゃんとローカライズもされている。

[shell gutter=“false”] $ sudo add-apt-repository ppa:gnome3-team/gnome3 $ sudo apt-get update $ sudo apt-get install gnome-shell [/shell]

GDM でログインする際に「Ubuntu GNOME Shell Desktop」を選択する。

上部パネルの日付、時刻の書式はスクリプトをカスタマイズ。シャットダウン系のメニューもスクリプトなので後で改造しよう。

MeeGo Tablet を使ってみた

2011年04月06日

Inspiron duo に入れたIntel 版のMeeGo Tablet が動くようになった。

隠しアクセスポイントに接続するには、一旦端末で、 [shell gutter=“false”]

iwconfig wlan0 essid key

[/shell] とするとWi-Fi の設定画面のアクセスポイント一覧に追加される。

画面のキャプチャは [shell gutter=“false”] $ xwd -root > ScreenShot.xwd [/shell]

画面の切り替えは、Windows キー。パネル画面（横）、すべてのアプリケーション画面（縦）、実行中のアプリケーションの選択などができる。

パネルの配置を変えるには、タブの右上隅をドラッグ。パネルの設定を変えるには、タブをシングルクリック（パネルが回転する）

標準で再生可能な形式はogg/ogv 。ffmpeg, ffmpeg2theora で変換すると簡単。

Inspiron Duo でALSA

2011年04月03日

http://ubuntuforums.org/showthread.php?t=1658635

Inspiron duo にLinux をインストールしてイヤフォンから音が出ないので調べてみたら、Ubuntu のフォーラムに解決方法が掲載されていた。model をdell とかdell-laptop にしても治らなかったのに、ideapad で解決するとは…

[shell gutter=“false”] $ sudo gedit /etc/modprobe.d/alsa-base.conf [/shell]

options snd-hda-intel model=ideapad

Fedora の場合は、/etc/modprobe.d/dist-alsa.conf を修正。

他にもタッチスクリーンでのマルチタッチの設定とか、画面回転の設定とかがあるので試してみよう!!

Inspiron Duo でLinux

2011年04月02日

Ubuntu 11.04b1

natty-desktop-i386.iso 普通に動いた。イヤフォンを繋いでも内臓スピーカーが動作してしまうのは後で調べよう。

Fedora 15a

Fedora-15-Nightly-20110401.17-i686-Live-desktop.iso 普通に動いた。イヤフォンが優先されないのは同様。

MeeGo Tablet 1.2 のIntel 版

meego-tablet-ia32-pinetrail.iso ライブイメージで動作した。タッチパネルは使えたが、タッチスクリーンが使えなかった。イヤフォンが優先されないのは同様。アクセスポイントのリストは公開されているものが表示される。隠しアクセスポイントに接続するインタフェースがない？メニューを戻る操作がよくわからなかった。

MeeGo Tablet 1.2 の本家版

meego-tablet-ia32-pinetrail-1.1.99.0.20110330.3.img 縦モードで起動。壁紙がかっこよい。タッチスクリーンが動作せず。残念。

MeeGo Netbook 1.2 の本家版

meego-netbook-ia32-1.1.99.0.20110330.3.img 普通に動作。ソフトウェアリポジトリが貧弱なような気がする。標準でNTFS が参照不可。Fedora 版かopenSUSE 版に切り替えた方がよいかも。

Android-x86 2.2

android-x86-2.2-sparta.iso Inspiron Duo 用のビルドなだけに、よく動く。画面表示もネットワークも日本語表示もOK。イヤフォンが優先されないのは他と同様。
