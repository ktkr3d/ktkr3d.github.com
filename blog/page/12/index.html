HAIKU R1 A3を試してみた

2011年06月22日

http://www.haiku-os.org/news/2011-06-18_haiku_release_1_alpha_3

BeOS みたいなOS、HAIKU Release 1 Alpha 3 がリリースされたので試してみた。 Linux でもBSD でもない独自カーネル。 ライブUSB の状態で、OGG、MP3、MP4(H.264+AAC)の再生が可能。 NTFS、EXT4 に対する読み/書きマウントが可能。 ビデオデバイス、サウンドデバイスを特別な設定なしに自動認識して正常動作したのは、何気にすごいことかもしれない。Ubuntu 11.04 ですら設定変更が必要だったのに。

Android-x86 3.1 らしきもの

2011年06月16日

http://groups.google.com/group/android-x86/browse_thread/thread/6077280a8b65e525

Android-x86 3.1 らしきものがあったので、実機(Inspiron duo)とVirtual Box の両方で動かしてみた。HDMI がうまく動かなかったので、VESAモードで起動。GPL 版でUI は2系のままらしい。ターミナルは使用可能。 実機では縦モードになった。レスポンスは軽快。

Peercast Radio をMac で動かしてみた

2011年06月15日

http://ktkr3d.site11.com/?page_id=11

Firefox でPeercast の配信を選局するアドオン「Peercast Radio for Firefox」をMac OS X 10.6.7 で試してみたら、改造なしで動作した。Firefox のアドオンだし、難しいことをしていないから当然か。 Mac OS X 用のPeercast 本体はCocoaYP さんのビルドをお借りした。外部プレイヤーにはVLC Media Player を選び、外部プレイヤー設定画面の「外部プレイヤー1(MMSストリーム)」を以下のように設定した。

  • 場所
  • 「mms: の代わりにmmsh: を使用する」をチェック！

Linux と同様にmmsh: にすると再生待ちが発生しないみたい。 Quick Time Player のパスには空白が入るから、少し修正が必要そう。

最近の更新は、以下のとおり。

  • 0.1.0a5 (2011/05/24) Firefox 5.0 対応

  • 0.1.0a4 (2011/02/12) Firefox 4.0 対応、Ustream イエローページ追加
