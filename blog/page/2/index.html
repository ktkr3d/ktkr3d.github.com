Ubuntu 12.10 + Calipso CMS (node.js, Express, MongoDB)

2013年01月12日

https://github.com/cliftonc/calipso

Calipso はnode.js(サーバサイドJavaScript)、express(node.js 上のフレームワーク)、MongoDB で構築されたシンプルなCMS アプリです。node.js のお試し用にUbuntu 12.10 上でCalipso 0.3.17 の環境を構築してみました。

  1. パッケージのインストール Ubuntu オフィシャルリポジトリのnode.js はバージョンが古いのでPPA のリポジトリを追加します。 [shell gutter=“false”] $ sudo add-apt-repository ppa:chris-lea/node.js
    $ sudo apt-get update
    $ sudo apt-get install nodejs npm mongodb [/shell]

  2. Calipso のインストールの前に このままの状態でCalipso のインストールを進めると、node コマンドが見つからない旨のメッセージを出力して、途中で終了してしまいます。Debian/Ubuntu には別にnode コマンドがあるため、node.js のコマンドはnodejs になっているようです。暫定的にシンボリックリンクを作成しておきます。作成したシンボリックリンク(/usr/bin/node)は後で消しておきます。 [shell gutter=“false”] $ sudo ln -s /usr/bin/nodejs /usr/bin/node [/shell]

  3. Calipso のインストールと起動 [shell gutter=“false”] $ sudo npm install -g node-gyp $ sudo npm install -g calipso $ sudo calipso site /var/www/MySite $ cd /var/www/MySite $ sudo calipso server [/shell]

  4. ブラウザからアクセス http://localhost:3000 表示されたウィザードに従って、データベースの作成、アカウントの作成をします。表示に関しては、一応、日本語設定が用意されています。

  5. nginx からリバースプロキシで接続 [shell gutter=“false”] $ sudo apt-get install nginx [/shell]

[plain title=“/etc/nginx/sites-available/default” firstline=“30” highlight=“33,36”]

location / {
    # First attempt to serve request as file, then
    # as directory, then fall back to displaying a 404.
    #try_files $uri $uri/ /index.html;
    # Uncomment to enable naxsi on this location
    # include /etc/nginx/naxsi.rules
    proxy_pass http://localhost:3000/;
}

[/plain]

[shell gutter=“false”] $ sudo /etc/init.d/nginx reload $ sudo /etc/init.d/nginx restart [/shell] http://localhost からアクセスできるようになります。

Moc 2.5.0 Beta1 + Patch R2519/r2520

2013年01月04日

http://moc.daper.net/node/850

moc (music on console) はコンソールで動作する軽量な音楽プレイヤーです。マルチバイト文字に依存してトリム位置がずれるバグがありましたが、解決(2.5.0 beta1 + patch r2519/r2520)されました。これで日本語のパスやタグが入っていてもレイアウトが崩れなくなりました。現時点でのUbuntu のリポジトリに存在するmoc のバージョンは2.5.0 alpha4 + svn20120224-1ubuntu1 ですので、最新のソースを取得して、コンパイル、インストールしてみました。

[shell gutter=“false”] $ sudo apt-get build-dep moc $ svn co svn://daper.net/moc/trunk $ cd trunk $ ./autogen.sh $ ./configure —prefix=/usr $ make -j4 $ sudo make install [/shell]

Thunderbird + GMail + Google Calendar + Google Tasks

2012年12月08日

Thunderbird とGoogle のメール、カレンダー、TODO を同期してみました。

Thunderbird のインストール

thunderbird をインストールします。

Thunderbird アドオンのインストール

Thunderbirdで、 [アドオン]メニューから下記のアドオンを検索してインストールします。

  • Lightning https://addons.mozilla.org/ja/thunderbird/addon/lightning/

  • Provider for Google Calendar https://addons.mozilla.org/ja/thunderbird/addon/provider-for-google-calendar/

  • Google Tasks Sync https://addons.mozilla.org/ja/thunderbird/addon/google-tasks-sync/

  • Google Contacts https://addons.mozilla.org/ja/thunderbird/addon/google-contacts/

メールアカウントの設定

Thunderbird で、

  1. [新規作成]–[既存のメールアカウント…]メニューを選択

  2. Google のアカウント情報を指定

Google カレンダーのアドレスの取得

Google カレンダーで、

  1. [設定]メニューを選択

  2. [カレンダー]タブをクリック

  3. [このカレンダーを共有]リンクをクリック

  4. [カレンダーの情報]タブをクリック

  5. カレンダーのアドレスの[XML]リンクを右クリックして[リンクのURLをコピー]

カレンダーの同期設定

Thunderbird で、

  1. [新規作成]–[カレンダー…]メニューを選択

    • カレンダーの保存先: ネットワークのサーバに保存する

    • フォーマット: Google カレンダー

    • 場所: 上記手順で取得したGoogle カレンダーのアドレス

    • オフラインサポート: チェック

    • パスワードマネージャにこのパスワードを保存する。: チェック

    • カレンダー名を設定

TODO の同期設定

Thunderbird で、

  1. 右下ペインの「Google Tasks Sync」の「Click here to get started.」をクリック

  2. アカウント情報を指定してログイン

  3. [アクセスを許可]をクリック
