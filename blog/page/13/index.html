GNOME Shell 拡張を作ってみよう

2011年05月14日

http://live.gnome.org/GnomeShell/Extensions http://blog.fpmurphy.com/2011/04/gnome-3-shell-extensions.html

まずは、上部のパネルをクリックすると一定時間メッセージを表示するHello World 的な物から！！とりあえず日本語はunescape() してみた。

拡張を格納するディレクトリ

~/.local/share/gnome-shell/extensions/hello-world@ktkr3d.site11.com/

extension.js ファイル

[plain] // Sample extension code, makes clicking on the panel show a message const St = imports.gi.St; const Mainloop = imports.mainloop;

const Gettext = imports.gettext.domain(‘gnome-shell-extensions’); const _ = Gettext.gettext;

const Main = imports.ui.main;

function _showHello() {

let text = new St.Label({ style_class: 'helloworld-label', text: _(unescape("%uFF0F%u4EBA%u25D5%u203F%u203F%u25D5%u4EBA%uFF3C%u300C%u50D5%u3068%u5951%u7D04%u3057%u3066%u9B54%u6CD5%u5C11%u5973%u306B%u306A%u3063%u3066%u3088%uFF01%uFF01%u300D")) });
let monitor = global.get_primary_monitor();
global.stage.add_actor(text);
text.set_position(Math.floor (monitor.width / 2 - text.width / 2), Math.floor(monitor.height / 2 - text.height / 2));
Mainloop.timeout_add(3000, function () { text.destroy(); });

}

// Put your extension initialization code here function main(metadata) {

imports.gettext.bindtextdomain('gnome-shell-extensions', metadata.localedir);

Main.panel.actor.reactive = true;
Main.panel.actor.connect('button-release-event', _showHello);

} [/plain]

metadata.json ファイル

[plain] { “uuid”: “hello-world@ktkr3d.site11.com”, “name”: “Hello, World!”, “description”: “An example extension to show how it works. Shows Hello, world when clicking on the top panel.”, “shell-version”: [ “3.0” ], “localedir”: “/usr/local/share/locale”, “url”: “http://ktkr3d.site11.com” } [/plain]

stylesheet.class ファイル

[plain] / Example stylesheet / .helloworld-label {

font-size: 36px;
font-weight: bold;
color: #ffffff;
background-color: rgba(10,10,10,0.7);
border-radius: 5px;

} [/plain] 拡張を有効にするには、[Alt]+[F2]でrを入力して[Enter]

PC-BSD を試してみた

2011年05月13日

http://www.pcbsd.org/

FreeBSD をインストールしてみたけれど、PC-BSD の方が何かと楽そうなので、PC-BSD 8.2 をインストールしてみた。インストール前に基本パーティションを作っておくと、インストールが楽かも。 [shell gutter=“false”]

uname -a [/shell]

FreeBSD **** 8.2-RELEASE FreeBSD 8.2-RELEASE #7: Wed Feb 16 12:19:08 PST 2011 root@build8x32.pcbsd.org:/usr/obj/usr/local_storage/pcbsd-build82/fbsd-source/8.2/sys/PCBSD i386

デスクトップ環境はKDE で、Firefox, Thunderbird, VLC 等をインストール時に追加できる。特に設定しなくても日本語も使えるし、mp3 も再生できる。GRUB からのチェインロードも問題無し。すごく頑張ってると思う。 Inspiron duo の1366x768 の解像度に設定すると、ディスプレイが表示されなくなってしまうので、とりあえずVESA で凌ごう。Linux と比べてドライバ(ディスプレイ、ファイルシステム)の情報は少ないかも。 ps コマンドやdf コマンドがBSD っぽくて少し懐かしい感じがした。

Hex-a-Hop で遊んでみた

BURG を試してみた

2011年05月07日

http://code.google.com/p/burg/ http://www.omgubuntu.co.uk/2011/05/beautiful-burg-boot-loader-gets-ubuntu-11-04-ppa/

ブートローダをBURG(Brand-new Universal loadeR from GRUB)にしてみた。 カラフルでいろんなテーマがあっておもしろいし、いちいちOS をリブートしなくても設定を試せるのでお手軽！

Android-x86 とMeeGo のアイコンがなかったので、適当に追加してみた。 *edit: gnome-look.org にあった!! http://gnome-look.org/content/show.php/Faenza-style+Linux+OS+icons+for+%22Burg%22?content=138684

  1. burg をインストール

  2. /etc/default/burg でテーマを設定 [plain highlight=“2”] GRUB_GFXMODE=1366x768 GRUB_THEME=fortune [/plain]

  3. 追加するOS のアイコンファイルを作成 カラーとグレイスケールのPNG ファイル（128x128, RGB モード）を用意して、テーマディレクトリ内のicons ディレクトリに格納

  4. 追加するOS に対応する定義をhover ファイルに追加 [plain firstline=“16” highlight=“17,18”] -slackware { image = “$$/grey_slackware.png:$$/large_slackware.png” } -android { image = “$$/grey_android.png:$$/large_android.png” } -meego { image = “$$/grey_meego.png:$$/large_meego.png” } } [/plain]

  5. /boot/burg/burg.cfg 内のmenuentry でクラスを指定 [plain firstline=“143” highlight=“143”] menuentry “MeeGo Tablet 1.2 (on /dev/sda9)” —class meego —class os —group group_/dev/sda9 { insmod ext2 set root=‘(hd0,9)’ search —no-floppy —fs-uuid —set d6dc4505-c05f-434a-af52-8cd3d543b03f linux /boot/vmlinuz-2.6.38.2-8.1-adaptation-pinetrail root=/dev/sda9 } [/plain]

  6. 試してみる！ [shell gutter=“false”] $ sudo burg-emu -D [/shell]

GNOME Shell を拡張してみた

2011年04月09日

https://wiki.archlinux.org/index.php/GNOME_3 https://live.gnome.org/GnomeShell/CheatSheet http://www.webupd8.org/2011/04/gnome-shell-extensions-additional.html http://git.gnome.org/browse/gnome-shell-extensions/

GNOME Shell Extensions やGNOME 3 の便利な機能を使ってみよう。

GNOME Shell Extensions

[shell gutter=“false”] $ git clone git://git.gnome.org/gnome-shell-extensions $ cd gnome-shell-extensions $ ./autogen.sh $ make $ sudo make install [/shell]

  • alternate-tab: Alt+Tab 操作関連

  • alternative-status-menu: 電源オフメニュー常時表示 標準ではステータスメニューを開いて、「Alt」ボタンを押すと「サスペンド」が「電源オフ…」に切り替わる。alternative-status-menu を使うと「サスペンド」と「電源オフ」の両方が常に表示されるようになり、「Alt」ボタンを押すと「サスペンド」が「Hibernate」に切り替わる。必須!!

  • dock: 右端にドックバーを表示（画面が狭くなった…微妙?）

  • windowsNavigator: overview 画面上でALT+数字キーによりウィンドウ選択

GNOME 3

  • テーマのカスタマイズ gnome-tweak-tool またはsetting.ini

  • スクリーンキャストを撮る Ctrl+Alt+Shift+R で開始&終了。WEBM 形式で出力される。

  • キー操作でoverview へ切り替える システム(Windows)キーまたはAlt+F1
